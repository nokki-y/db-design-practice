# db-design-practice

「達人に学ぶDB設計徹底指南書 第2版」（ミック著、翔泳社、2024年）で学んだ理論を、自分の手で実践するための演習リポジトリ。

書籍の内容をそのまま写すのではなく、独自の題材に対して設計を考え、思考過程を記録することで理解を深める。

## 進め方

1. 書籍の章を読んで理論を理解する
2. 独自の題材（非正規形テーブル等）に対して、自分で設計を考える
3. 思考過程（何に詰まったか、なぜそう判断したか）を NOTE.md に記録する
4. 必要に応じてSQLで検証する

## 進捗

| 章 | タイトル | 優先度 | 状態 |
|---|---|---|---|
| 第1章 | データベースを制する者はシステムを制す | - | スキップ（導入・概論のため演習対象外） |
| 第2章 | 論理設計と物理設計 | - | スキップ（座学中心でテーブル設計の演習に不向き） |
| 第3章 | 論理設計と正規化 | 高 | ✅️ |
| 第4章 | ER図 | - | スキップ（AIで生成可能なため演習対象外） |
| 第5章 | 論理設計とパフォーマンス（非正規化） | 高 | ✅️ |
| 第6章 | データベースとパフォーマンス | 低 | ✅️ |
| 第7章 | 論理設計のアンチパターン | 高 | - |
| 第8章 | 論理設計のグレーノウハウ | 中 | - |
| 第9章 | 一歩進んだ論理設計（木構造） | 中 | - |
| 付録 | 正規化 vs インデックス vs 非正規化 パフォーマンス比較 | - | ✅️ |

## 環境

- PostgreSQL 16（Docker Compose で起動）
- pgAdmin 4（GUI確認用）

```bash
docker compose up -d

# 接続（接続情報は .env.example を参照）
psql -h localhost -p 5555 -U postgres -d db_design
```
